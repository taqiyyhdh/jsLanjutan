JAVASCRIPT
>> single thread - non-blocking - asynchronous - concrurrent language
 

VARIABEL
pake let atau const: 
error terdeteksi
block scope (seperti bahasa pemrograman lainnya) bukan function scope

kalo pake cons:
sama aja seperti let tapi digunakan ketika ingin isi variabelnya tetap atau tidak berubah
seperti i++ itu tidak bisa pake const
digunakan untuk meminilimasir perubahan state

FUNCTION
js memperlakukan function sebagai object

- declaration >> function seperti pada umummnya

- expression (disimpan dalam sebuah variabel)

- arrow >> bentuk ringkas dari function expression
ex: fex penulisan function (nama) - far (nama) =>

- higher order >> function yang beroperasi pada function yang lain, 
baik itu digunakan dalam argumen (callback) maupun sebagai retun value
digunakan untuk menyederhanakan kode efektif efesien
ex: map, filter, reduce

TEMPLATE LITERAL
>> string literal yang memungkkinkan adanya expression di dalamnya
  string literal menggunakan '' "" ``
  string literal dapat di gunakan jika menggunakan back tip `` bisa memperlakukan
+ multi line string 
+ embedded expression 
+ HTML fragments
+ expression interpolation
+ Tagged template >> bentuk kompleks dari template literals
  memungkinkan kita untuk membaca template literals melalui sebuah function
  penggunaan lain:
  = escaping HTML tags
  = translation & internalization
  = styled componens

ASYNCHRONOUS JAVASCRIPT
1. CALLBACKS
  >> function yang dikirimkan sebagai parameter pada function yang lain
      function yang dieksekusi setelah fungsi lain selesai dijalankan
2. PROMISE
  >> object yang mempresentasikan keberhasilan/kegagalan sebuah event yang asynchronous di masa yang akan datang
    janji (terpenuhi / ingkar)
    states (fulfiled / rejected / pending)
    callback (resolve / reject / finally)
    aksi (then / catch)
3. AJAX
4. FECH
  >> sebuah method pada API pada js modern yang tugasnya mengambil 
  RESOURCE dari jaringan, dan mengembalikan PROMISE yang selesai (FULLFILLED)
  ketika ada RESPONSE yang tersedia
  fetch(resource (url/request object), init(konfigurasi tambahan));
5. ASYNC & AWAIT
  asynchronous function
  >> fungsi yang berjalan secara asynchronous
    menghasilkan (implisit) PROMISE
    cara penulisannya sama seperti synchronous function (function biasa)

EEROR HANDLING 
>> promise (.then() .catch())
   async await (try catch())
   fetch (try catch()-throw new error)
